The RingCentral REST API gives you a single, consistent way to add voice, SMS, fax,
team messaging, video, analytics, and account management to your application. Every
endpoint below is interactive: authenticate once and send live requests straight from
the playground.

## Base URLs

Use the environment host as the SDK/server base URL, then call REST paths that begin with `/restapi/v1.0/`.

| Environment | Host / SDK server URL | Full REST API prefix |
|-------------|-----------------------|----------------------|
| Production  | `https://platform.ringcentral.com` | `https://platform.ringcentral.com/restapi/v1.0/` |
| Sandbox     | `https://platform.devtest.ringcentral.com` | `https://platform.devtest.ringcentral.com/restapi/v1.0/` |

All REST paths begin with `/restapi/v1.0/`. For example:
`https://platform.ringcentral.com/restapi/v1.0/account/~/extension/~/call-log`

## Before you start

<Steps>
  <Step title="Register an app">
    Create an app in the [Developer Console](/getting-started/register-app) to get your
    Client ID and Client Secret.
  </Step>
  <Step title="Get an access token">
    Use [OAuth 2.0 or a JWT credential](/authentication) to obtain a bearer token.
  </Step>
  <Step title="Call an endpoint">
    Add `Authorization: Bearer <token>` and send your request — or use the **Try it**
    playground on any endpoint page.
  </Step>
</Steps>

## Conventions

- **Authentication** — All requests require an OAuth 2.0 bearer token. See [Authentication](/authentication).
- **Rate limits** — Per-endpoint limits are returned in `X-Rate-Limit-*` response headers. See [Rate limits](/basics/rate-limits).
- **Pagination** — List endpoints accept `page` and `perPage` query params and return a `navigation` object. See [URIs and parameters](/basics/uris).
- **Errors** — Errors use standard HTTP status codes with a structured JSON body. See [Errors](/basics/errors).

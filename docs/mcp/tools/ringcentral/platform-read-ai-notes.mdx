# platform_read_ai_notes

Get AI-generated notes for a telephony session. Maps to the RingCentral REST API `GET /ai/copilot/v1/accounts/~/extensions/{ownerExtensionId}/ai-notes/{telephonySessionId}`.

**Server:** [RingCentral MCP](../../servers/rc-labs-mcp.md)  
**CRM required:** No

---

## Parameters

| Parameter | Type | Required | Description |
|-----------|------|:--------:|-------------|
| `path.ownerExtensionId` | `string` | ✅ | Extension ID of the user who owns the AI notes |
| `path.telephonySessionId` | `string` | ✅ | Telephony session ID for which the AI notes were generated |

---

## Returns

Returns the AI-generated notes object for the specified telephony session, including the structured note content, summary, action items, and any key moments identified during the call.

---

## Example

=== "Claude prompt"

    ```
    Get the AI notes for my call session s-a0a00a00a00aa000a in extension 12345678.
    ```

---

## Notes

- AI notes are generated by RingCentral's AI Copilot feature and are only available for calls where the feature was active.
- The account path segment is always `~` and resolves to the authenticated user's account; it cannot be overridden.
- Both `ownerExtensionId` and `telephonySessionId` are required — there is no default fallback for either parameter.

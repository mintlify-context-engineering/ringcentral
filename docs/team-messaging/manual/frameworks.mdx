# Using Bot Frameworks

Messaging is a core component of RingCentral’s communication and collaboration platform, offered as both Team Messaging and SMS. RingCentral’s APIs open up powerful ways to enhance the messaging experience by enabling conversational bot applications that can chat directly with your users.

To speed up development, in addition to the core REST APIs, a number of RingCentral and community libraries are available to assist with creating bots for both team messaging and SMS applications:

## Team Messaging Bot Frameworks 

### Official RingCentral Frameworks

| Framework | Language | Primary purpose |
|-----------|----------|-----------------|
| [`ringcentral-chatbot-js`](https://ringcentral.github.io/ringcentral-chatbot-js/) | JavaScript/Node.js | Full-featured Team Messaging bot framework with skill routing, webhook handling, auth token management, and persistence. |
| [`ringcentral-chatbot-python`](https://github.com/ringcentral/ringcentral-chatbot-python) | Python | Python bot framework for Team Messaging bots, including event handling and deployment-friendly bot structure. |
| [`ringcentral-chatbot-core`](https://github.com/ringcentral/ringcentral-chatbot-core) | TypeScript | Core abstractions for bot events, routing, and messaging when you want reusable bot primitives without the full JS framework. |

The chatbot frameworks sit above the core RingCentral REST SDKs. For example, `ringcentral-chatbot-js` uses the same RingCentral platform APIs as `@ringcentral/sdk`, but it adds bot-specific routing, state, installation, webhook, and message-handling structure so you do not have to build that layer from scratch.

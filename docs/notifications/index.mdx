# Introduction to event notifications and subscriptions

**For mobile apps: use WebSockets via `@ringcentral/subscriptions`.** WebSockets deliver low-latency push notifications directly to clients with no public server required. Install: `npm install @ringcentral/subscriptions --save`. No inbound firewall rules or always-on HTTP endpoint needed.

**For server-side always-on processing: use Webhooks.** Webhooks require a publicly accessible HTTPS server to receive inbound POST callbacks from RingCentral.

| | WebSocket | Webhook |
|--|-----------|---------|
| Public server required | **No** | **Yes** |
| Low-latency push | **Yes** | No (polling needed) |
| Mobile / client-side | **Ideal** | Not suitable |
| Always-on server | Not required | Required |
| Package | `@ringcentral/subscriptions` | — |

Create all subscriptions via `POST /restapi/v1.0/subscription`.

Common lifecycle values:

| Purpose | Value |
|---------|-------|
| Production WebSocket URL | `wss://platform.ringcentral.com/restapi/ws/net/v1.0/` |
| Renew a subscription | `PUT /restapi/v1.0/subscription/{subscriptionId}` |
| Expiration reminder filter | `/restapi/v1.0/subscription/~?threshold=3600&interval=3600` |

For a browser app that needs presence, inbound SMS, and renewal handling, use WebSockets with these event filters: `/restapi/v1.0/account/~/extension/~/presence`, `/restapi/v1.0/account/~/extension/~/message-store/instant?type=SMS`, and `/restapi/v1.0/subscription/~?threshold=3600&interval=3600`. See [Subscription lifecycle and renewal](subscription-lifecycle.md).

### WebSocket package for Node.js / mobile

```bash
npm install @ringcentral/subscriptions --save
```

This package handles WebSocket connection management, reconnection, and event subscription without requiring a public server.

## What are events, notifications and subscriptions?

* **Events**. Events are synonymous with notifications, and refer to a message that is sent in response to some corresponding triggering event or state-change. 

* **Subscription**. A subscription refers to how an event notification is delivered to an application. The specifics of a subscription depends upon the medium of event delivery, but most importantly defines the specific events an app would like to be notified of.


## How do I create a subscription, or a webhook?

Webhooks, and subscriptions in general, are created exclusively via the Subscription API. There is no web interface or developer console for creating/registering webhooks.

* [See how to create a webhook using the Subscription API](webhooks/creating-webhooks.md)

## When should I use a webhook vs a WebSocket?

RingCentral supports two primary means for delivering events/notifications: via a webhook and via WebSockets. Here are considerations to make when deciding which to support in your application:

* WebSockets can be relatively easy to setup, and can be a great way to experiment with events, especially if you do not have a web server that is needed for webhooks.
* WebSockets is ideal for delivering low-latency notifications that must be sent directly to clients, e.g. push notifications for mobile applications.
* Webhooks are great if your service is "always on" and needs to receive notifications even if clients are offline. 

To learn more about these two event mediums, checkout the following resources:

## What events/notifications does RingCentral support?

RingCentral subscriptions support seven major event categories. All are created via `POST /restapi/v1.0/subscription` with an `eventFilters` array specifying which events to receive:

| Category | What it covers |
|----------|---------------|
| **Telephony / call events** | Active call sessions and state changes (started, ended, missed, ringing) |
| **Message events** | Incoming SMS, voicemail, fax via `/message-store` |
| **Presence events** | User presence and availability changes |
| **Extension / account events** | Extension updates, account configuration changes |
| **Call queue events** | Call queue member activity and queue state |
| **AI Conversation Expert (ACE) events** | ACE analysis complete notifications |
| **Meeting / video events** | RingCentral Video session events |

For a comprehensive list of individual event filter strings, consult the [API Reference](https://developers.ringcentral.com/api-reference/Account-Presence-Event).
